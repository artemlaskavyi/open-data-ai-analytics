# Аналіз результатів моніторингу заробітної плати в Україні

**Мета:** Дослідити динаміку номінальної заробітної плати працівників у різних галузях економіки.

**Джерело:** [Портал відкритих даних (data.gov.ua)](https://data.gov.ua/dataset/412a6bfc-3ff9-4ba8-a35a-4a57166e6185)
**Файл:** `monzp102019.xlsx`

### Гіпотези:
1. Яка галузь продемонструвала найбільший приріст номінальної зарплати за період моніторингу?
2. Чи є суттєва різниця в оплаті праці між промисловими галузями та сферою послуг?
3. Які регіони України мають найвищу та найнижчу середню заробітну плату?

## Підсумки проєкту
Результати дослідження: виявлено лідерство фінансового сектору за рівнем зарплат та значний відрив м. Києва від інших регіонів.
Результати перевірки якості: дані очищено від артефактів Excel, структурні аномалії в колонках мінімальної зарплати ідентифіковано та задокументовано.

## Лабораторна: Контейнеризація в Docker

Цей проєкт є виконанням лабораторної роботи з контейнеризації модулів за допомогою Docker. Він включає завантаження даних, перевірку якості, аналітику та візуалізацію даних про заробітну плату, об’єднаних у єдину мікросервісну архітектуру з використанням Docker Compose та PostgreSQL.

### Структура мікросервісів
Кожен модуль розміщено в окремій теці зі своїм `Dockerfile` та залежностями `requirements.txt`:
- `db/` — використання офіційного образу `postgres:15-alpine` (база даних).
- `data_load/` — сервіс підготовки даних. Завантажує Excel, обробляє в DataFrames та записує у БД `PostgreSQL`.
- `data_quality_analysis/` — сервіс перевірки якості даних. Вичитує дані з БД, виявляє аномалії та зберігає звіт (у Markdown) в спільний Docker-том: `reports_vol`.
- `data_research/` — сервіс дослідження. Рахує статистики, перевіряє гіпотези (з БД) та формує JSON-файл з висновками в том `reports_vol`.
- `visualization/` — сервіс візуалізації. Будує графіки та зберігає їх у спільний том `plots_vol`.
- `web/` — Flask-сервіс (з гарним інтерфейсом), який відображає дані з БД та збережені у томах звіти й картинки. 

### Запуск 
Щоб локально запустити всі сервіси, встановіть Docker Workspace і виконайте команду у корені проєкту:

```bash
docker compose up --build
```

Система спочатку підніме базу даних (`db`), зачекає на її готовність. Потім запустить модуль `data_load`. Після його успішного відпрацювання паралельно запустяться сервіси аналітики та візуалізації. І лише після них запуститься `web`-сервер.

### Взаємодія та Порти
- **5000** - Веб-інтерфейс (доступний за адресою [http://localhost:5000](http://localhost:5000))
- **5432** - Внутрішній порт PostgreSQL (мережа `app-network`).
Дані між `data_load` та аналітичними модулями передаються через базу даних PostgreSQL. Результати роботи аналітичних модулів передаються до веб-сервісу через спільні Docker томи.

## Розгортання в Microsoft Azure (Terraform + Cloud Shell + cloud-init)

Цей проєкт налаштовано для автоматичного розгортання в хмарі Azure без необхідності мати локальне середовище за допомогою інструменту **Terraform**.

### Інструкція з розгортання

1. **Відкрийте Azure Cloud Shell**
   - Увійдіть на портал [Azure Portal](https://portal.azure.com/).
   - Натисніть на іконку **Cloud Shell** (символ `>_` у верхньому правому куті).
   - Оберіть **Bash** середовище.

2. **Завантажте проєкт**
   - У Cloud Shell виконайте клонування репозиторію:
     ```bash
     git clone https://github.com/artemlaskavyi/open-data-ai-analytics.git
     cd open-data-ai-analytics/infra/terraform
     ```

3. **Розгорніть інфраструктуру**
   - Ініціалізуйте Terraform:
     ```bash
     terraform init
     ```
   - Запустіть застосування конфігурації:
     ```bash
     terraform apply
     ```
   - Введіть `yes`, коли Terraform запитає підтвердження.
   - Terraform створить ресурсну групу, мережу, віртуальну машину (Linux VM) та інші необхідні компоненти. Налаштування віртуальної машини та запуск Docker контейнерів відбувається автоматично за допомогою переданого сценарію `cloud-init.yaml`.

4. **Отримайте результати та перевірте роботу**
   - Після успішного виконання команди, Terraform виведе в термінал публічну IP-адресу віртуальної машини та посилання на веб-інтерфейс:
     ```text
     Outputs:
     public_ip_address = "xx.xx.xx.xx"
     web_interface_url = "http://xx.xx.xx.xx:5000"
     ```
   - Скопіюйте `web_interface_url` та відкрийте в браузері.
   - Оскільки Docker завантажує образи, встановлює залежності та ініціалізує базу даних, інтерфейс може бути доступним не одразу, а через 1-3 хвилини після створення віртуальної машини.
